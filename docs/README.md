# Docs

Project documentation lives in this directory.

- `task-management-philosophy.md`: workflow and product rationale
- `data-model.md`: runtime item model, workflow fields, and vault mapping
- `frontmatter-vault-model.md`: target storage model using per-item frontmatter and rebuildable indexes
- `storage-model.md`: persistence format, note/log separation, and archive rules
- `issue-driven-knowledge-workbench.md`: next-generation vault model for inbox, task, issue, theme, and knowledge workflows
- `ui-dual-view-design.md`: unified workbench design for plan and action lanes on one screen
- `ui-rendering-notes.md`: common TUI rendering failures, causes, and safe fixes
